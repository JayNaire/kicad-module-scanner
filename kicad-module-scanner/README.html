kicad_module_scanner


A utility to scan, view and copy Kicad footprint (module) libraries.

Requires:
Qt sdk (qmake, uic etc.) http://qt.nokia.com/downloads

A Postscript viewer - okular preferred
gawk (GNU awk)
bash + the standard GNU core utilities C++ compiler

To compile:
If not using Eclipse then:

cd /wherever/you/unpacked/the/archive
check that kicad_module_scanner.pro is in the current directory (if it isn't you may need to change to a subdirectory)
qmake
make -f Makefile.Release

should produce an executable called kicad_module_scanner

Test run:
cd /wherever/you/unpacked/the/archive
./kicad_module_scanner
On the General Setup tab the only thing to check is the location of the Postscript viewer.

On the File Scanning tab add a file or files to scan.
Set the "Output to Database File" to whatever you like.
Press "Scan Checked Files"
If all goes well the output file you specify will be created.

Switch to the Viewing tab.
Check the "Footprint Database" file is the same as you specified above.
Check no search criteria are specified so you see all the modules in the library.
Click Search.
The Search Results list should fill up.
Click an item in the list.
Okular (or whatever you set in General Setup) should spring in to life and display the selected footprint. You may need to faff around with the viewer to get the best presentation mode, zoom etc.

Once you get comfortable with the basic idea of kms you can try scanning whole folders of mod files on the Folder Scanning tab.

Warnings:
Some features are not fully tested and may fall over with "strange" mod files. I have discovered that (very) old footprints used different scaling from the "tenth of a mil" used now. kms knows nothing about them so be warned! Pins without copper annular rings are holes but are not properly dealt with.

Postscript Viewer Notes:
Pin 1 is always drawn in red - but holes (see above) may mean you don't see them as when Pin 1 is actually, say, a heatsink mounting hole. Pin Pitch dx, dy dimensions should be accurate; "Footprint XxY" are might be wrong sometimes (untested) If refdes or module names look daft, don't blame kms. Some mod files on the web are indeed daft. Some footprints converted from other PCB programs do not get pin rotation correct so they look wrong in kms. (They look wrong in Kicad too.) You have to use Kicad's module editor to sort them out.

Enjoy!.
